# 小红书发图推广方案（Roam Focus / “专注即漫游”）

> 目标：引导 iOS 用户下载/注册（适配：学生备考 + 职场打工人）

---

## 1. 我对项目的理解（用于提炼卖点）

**Roam Focus** 是一款 iOS 专注计时器，但核心差异点不是“番茄钟”，而是把**专注时间变成一段“真实地图上的虚拟漫游”**：

- **三种交通模式**：步行 / 骑行 / 驾驶（不同速度 → 不同时长的专注场景）
- **真实世界地图路线**：基于当前位置随机目的地 + MapKit 路线；头像沿路线移动
- **探索反馈**：沿途 **POI 兴趣点发现**（餐饮/地标/公园等）+ Toast 提示
- **过程控制**：暂停/继续；停止可保存记录
- **长期激励**：历史记录/统计；**奖杯系统（129个）**；（可选）天气/主题/语言实时切换
- **视觉风格**：Liquid Glass（玻璃态 + 动态渐变）——适合“发图种草”

这决定了小红书内容的主叙事应该是：  
**“专注不再是倒计时，而是地图上的一段冒险”**（从无聊→有反馈→有收集→可坚持）。

## 3. 发图结构：9宫格“高转化”排序（建议 7–9 张）

### 第2张（痛点对比）

- 左：传统番茄钟（你可用一张普通倒计时的截图/示意）

1) 选交通方式 + 时长

### 第4张（过程爽点：轨迹 + 头像移动）
**截图内容**：Active Roam 地图页，必须出现：

### 第6张（控制感：暂停/继续/停止保存）

## 4. 截图需要美化吗？结论：**“轻量美化 + 保留真实UI”**

### 为什么不建议“直接原图堆上去”
- 小红书信息流竞争强，**原图缺少信息引导**（用户不知道你想让他看什么）
- 原生截图字太小、重点不突出，收藏率/停留时间会差

### 为什么不建议“过度美化”
- 工具类/效率类内容最怕“像广告”，过度包装会降低信任与转化

### 推荐的“轻量美化”规范（最省事也最有效）
1. **统一画幅**：3:4（竖图）或 4:5（更适合信息流）；每张留同样边距  
2. **加设备框**：iPhone 机型框（白/黑两套），瞬间提升质感  
3. **每张只讲 1 个点**：顶部一句话标题 + 1–2 个箭头/高亮圈  
4. **关键数据放大**：时长/距离/奖杯数量用大号数字贴纸  
5. **隐私处理**：如果出现真实位置、定位点、家/公司附近地名，做模糊/遮挡  
6. **颜色体系**：沿用 App 的蓝紫/橙粉渐变做标题条/角标，保持品牌统一

### 工具链（任选其一）
- 最快：小红书自带拼图 + 标注 + 贴纸（即可用）
- 更好看：Canva / 创客贴（套模板做封面&排版）
- 更高级：Figma（做一套可复用模版，后续换截图即可复投）

## 5. 拍摄/截屏建议（避免翻车）

1. **尽量用真机录屏 → 抽帧**：地图动画/POI toast 更容易截到“爽点瞬间”
2. **用模拟定位**：设置一个“公共地点”作为起点（避免暴露隐私）
3. **浅色/深色各一套**（可选）：做第二篇内容时换主题即可“换皮复投”
4. **保持电量/信号/时间干净**：状态栏尽量统一（必要时后期裁掉）

## 7. 发布后“拿流量”的运营动作（决定你能不能起量）

### 0–2小时（最关键）
- 置顶评论：`想要下载链接/内测名额：评论【漫游】`
- 互动引导：正文末尾提问  
  - 学生：`你复习最容易分心的时段是？`  
  - 职场：`你一天能做几段深度工作？`
- 及时回复评论（尤其是前 20 条）：提升互动率与二次分发

### 24小时内
- 观察数据：曝光→点击→收藏→评论  
- 若收藏高、评论低：在评论区补“使用技巧/快捷入口/隐私说明”

### 复投节奏（建议 7 天做 4–6 篇）
1. 第1篇：总览种草（9图模板）
2. 第2篇：学生场景（背书/刷题 25/45min 组合）
3. 第3篇：职场场景（写方案/读论文 60–90min）
4. 第4篇：奖杯收集/打卡挑战（“7天漫游挑战”）
5. 第5篇：地图+POI 的“旅行感”合集（更偏审美）
6. 第6篇：FAQ（定位权限/耗电/隐私/适配机型）
